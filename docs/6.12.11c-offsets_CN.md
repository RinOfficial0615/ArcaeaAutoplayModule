# 偏移 / 备注（6.12.11c，ARM64）

Language: [English](6.12.11c-offsets.md) | 简体中文

本模块对 Arcaea 6.12.11c 的 `libcocos2dcpp.so` 进行了硬编码。

## Hook 点

逐帧逻辑（已 Hook）：

- `Gameplay_processLogicNotes` @ `0x147671C`

该 Hook 还会通过调用每个 note 的 `setBeingTouched` 虚表处理器（`+0x60`，传入伪造 `TouchLike` 结构）来驱动长条（hold/arc）高亮与粒子表现。

分数更新：

- `ScoreState_applyJudgement` @ `0x0ACEEF4`
- `ScoreState_applyMiss` @ `0x0E532B0`

模块会 Hook `ScoreState_applyJudgement` 并强制 `PERFECT_PURE`，避免在由伪造 hold/arc 触摸间接触发判定时出现 FAR/LOST。

## 触摸分发器（仅 RE 记录）

单例/持有者：

- `Cocos2dxTouchDispatcherHolder_getInstance` @ `0x0B141AC`
- `holder+0xF0` -> dispatcher 指针

dispatcher 虚表槽位：

- `+0xE0` began
- `+0xE8` moved
- `+0xF0` ended
- `+0xF8` cancel（未使用）

调用形式：

```c
typedef void (*DispatchTouches_t)(
    void *dispatcher,
    int count,
    int64_t *ids,
    float *xs,
    float *ys,
    float *zs,
    float *sizes,
    int64_t timestamp);
```

说明：当前自动游玩实现没有通过该 dispatcher 注入，因为在 Hook `Gameplay_processTouchInput` 时观察到了副作用。

## Note 扫描

Gameplay 上待处理 note 向量：

- `gameplay+0xA0` begin
- `gameplay+0xA8` end

通用 note 字段：

- `note+0x18` timeStart（ms）
- `note+0x1C` timeEnd（ms）
- `note+0x20` pos*（NotePosition）
- `note+0x40` playSceneOrCtx*（指针）
- `note+0x54` active（u8）

Arc 运行时字段：

- `note+0xCC` 运行时 x 偏移（float）
- `note+0xD0` 运行时 y（float）

Arc 标志：

- `note+0x9C` `isVoid`（i32）
  - `!= 0` => Trace/Void arc 段（不触摸、无判定）
- `note+0xC8` `activeNow`（u8）
  - 命中检测用它判断该 arc 当前是否可触摸

`LogicArcNote_setBeingTouched` 还会调用 `playSceneOrCtx` 上的一个虚函数：

- `(*(void (**)(void *))( *(uintptr_t *)playSceneOrCtx + 0x530 ))(playSceneOrCtx)`

如果 `playSceneOrCtx` 失效（例如跨谱面），可能崩溃；应将其视为安全门。

Hold 标志：

- `note+0xA0` `headActivated`（u8）
  - 由 `Gameplay_tryTapJudgementForTouch` 在点击 hold 头时置为 `1`
  - `Gameplay_processTouchInput` 在调用 `LogicHoldNote_setBeingTouched` 前会检查该值

Arc 子节点（arctap）：

- `arc+0x118` begin
- `arc+0x120` end

## 坐标

Arc 命中检测使用的实用映射：

- `base_x_world = x_norm * 850 - 425`
- `x_world = base_x_world + runtime_x`
- `y_world = runtime_y`

伪造触摸 NDC：

- `ndc_x = x_world / 425`
- `ndc_y = y_world / 275 - 1`

## Flick 手势检查

`Gameplay_processTouchInput` 会传入：

- `touch_sys_id = *(int *)(touch + 0x0C)`
- `touch_world_xy`（ray/plane 交点）

到 `LogicFlickNote_checkFlickGesture`（`0x12B80E8`）。该检查要求：

- 滑动距离 > `106.0`（世界坐标单位）
- 滑动方向与期望向量夹角在 `45` 度内

## Miss 窗口补丁（Autoplay）

本模块可补丁 `Gameplay_processLogicNotes`，让 tap/flick note 在到时后立即进入 miss 路径（偏移窗口改为 `0ms`）。随后由 `ScoreState_applyMiss` Hook 把 miss 转为 `PERFECT_PURE`。

- `0x1476C04`：`add w8, w10, #0x64` -> `add w8, w10, #0`
- `0x1476CBC`：`add w8, w10, #0x64` -> `add w8, w10, #0`
- `0x1476D0C`：`add w8, w10, #0xC8` -> `add w8, w10, #0`

## 长条：setBeingTouched（Autoplay）

`LogicArcNote` 里面有触摸处理使用的虚函数：

- 虚表槽位 `+0x60`（调用形式 `fn(self, TouchLike *touch, int now_ms)`）

`LogicHoldNote` 也有虚表 `+0x60` 处理器，但当前映射实现仅播放音效，并设置长条触摸标志（`*(u16*)(note+0x5C)=0x0101`）。

模块行为：

- Arc：每帧调用虚表槽位 `+0x60`（需要 `TouchLike.touch_uid`）。
- Hold：先设置 `*(u8*)(hold+0xA0) = 1`（head activated），并在 `timeStart < now < timeEnd` 期间每帧调用虚表槽位 `+0x60`。
  - 这会在起始时播放一次头部 SFX（AudioManager 基于 `timeStart` 去重），并设置 `*(u16*)(hold+0x5C)=0x0101`。
- 跳过 arc trace/void arc（`note+0x9C != 0`），避免其发粒子或进入判定。

stub 中使用的 `TouchLike` 字段：

- `touch+0x0C` system id（int）
- `touch+0x1C` ndc_x（float）
- `touch+0x20` ndc_y（float）
- `touch+0x34` touch_uid（int）
- `touch+0x3C` timestamp（int）
