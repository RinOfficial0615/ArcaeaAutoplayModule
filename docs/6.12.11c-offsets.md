# Offsets / Notes (6.12.11c, ARM64)

Language: English | [简体中文](6.12.11c-offsets_CN.md)

This module is hardcoded to Arcaea 6.12.11c `libcocos2dcpp.so`.

## Hook Points

Per-frame logic (hooked):

- `Gameplay_processLogicNotes` @ `0x147671C`

This hook also drives long-note (hold/arc) highlight + particles by calling each
note's `setBeingTouched` vtable handler (`+0x60`) with a synthetic `TouchLike` stub.

Score update:

- `ScoreState_applyJudgement` @ `0x0ACEEF4`
- `ScoreState_applyMiss` @ `0x0E532B0`

This module hooks `ScoreState_applyJudgement` and forces `PERFECT_PURE` to avoid
FAR/LOST when judgement is driven indirectly by synthetic hold/arc touches.

## Touch Dispatcher (RE only)

Singleton/holder:

- `Cocos2dxTouchDispatcherHolder_getInstance` @ `0x0B141AC`
- `holder+0xF0` -> dispatcher pointer

Dispatcher vtable slots:

- `+0xE0` began
- `+0xE8` moved
- `+0xF0` ended
- `+0xF8` cancel (unused)

Call form:

```c
typedef void (*DispatchTouches_t)(
    void *dispatcher,
    int count,
    int64_t *ids,
    float *xs,
    float *ys,
    float *zs,
    float *sizes,
    int64_t timestamp);
```

Note: current autoplay implementation does not inject through this dispatcher
because of side-effects observed when hooking `Gameplay_processTouchInput`.

## Note Scan

Pending note vector on gameplay:

- `gameplay+0xA0` begin
- `gameplay+0xA8` end

Common note fields:

- `note+0x18` timeStart (ms)
- `note+0x1C` timeEnd (ms)
- `note+0x20` pos* (NotePosition)
- `note+0x40` playSceneOrCtx* (pointer)
- `note+0x54` active (u8)

Arc runtime:

- `note+0xCC` runtime x offset (float)
- `note+0xD0` runtime y (float)

Arc flags:

- `note+0x9C` `isVoid` (i32)
  - `!= 0` => Trace/Void arc segment (do not touch, no judgement)
- `note+0xC8` `activeNow` (u8)
  - used by hit-test to gate whether the arc is currently touchable

`LogicArcNote_setBeingTouched` also calls a virtual on `playSceneOrCtx`:

- `(*(void (**)(void *))( *(uintptr_t *)playSceneOrCtx + 0x530 ))(playSceneOrCtx)`

If `playSceneOrCtx` is stale (e.g., cross-chart), this can crash; treat it as a safety gate.

Hold flags:

- `note+0xA0` `headActivated` (u8)
  - set to `1` by `Gameplay_tryTapJudgementForTouch` when the hold head is tapped
  - checked by `Gameplay_processTouchInput` before calling `LogicHoldNote_setBeingTouched`

Arc children (arctaps):

- `arc+0x118` begin
- `arc+0x120` end

## Coordinates

Practical mapping used by arc hit-test:

- `base_x_world = x_norm * 850 - 425`
- `x_world = base_x_world + runtime_x`
- `y_world = runtime_y`

Synthetic touch NDC:

- `ndc_x = x_world / 425`
- `ndc_y = y_world / 275 - 1`

## Flick Gesture Check

`Gameplay_processTouchInput` passes:

- `touch_sys_id = *(int *)(touch + 0x0C)`
- `touch_world_xy` (ray/plane intersection)

Into `LogicFlickNote_checkFlickGesture` (`0x12B80E8`). The check requires:

- swipe distance > `106.0` (world units)
- swipe direction within `45` degrees of expected vector

## Miss-Window Patch (Autoplay)

This module can patch `Gameplay_processLogicNotes` so tap/flick notes enter the
"miss" path immediately when their note time is reached (offsets become `0ms`).
The `ScoreState_applyMiss` hook then turns those misses into `PERFECT_PURE`.

- `0x1476C04`: `add w8, w10, #0x64` -> `add w8, w10, #0`
- `0x1476CBC`: `add w8, w10, #0x64` -> `add w8, w10, #0`
- `0x1476D0C`: `add w8, w10, #0xC8` -> `add w8, w10, #0`

## Long Notes: setBeingTouched (Autoplay)

`LogicArcNote` exposes a virtual handler used by touch processing:

- vtable slot `+0x60` (called as `fn(self, TouchLike *touch, int now_ms)`)

`LogicHoldNote` also has a vtable `+0x60` handler, but its mapped implementation
only plays SFX and sets the long-note touch flag (`*(u16*)(note+0x5C)=0x0101`).

Module behavior:

- Arc: call vtable slot `+0x60` each frame (needs `TouchLike.touch_uid`).
- Hold: set `*(u8*)(hold+0xA0) = 1` (head activated) and call vtable slot `+0x60` each frame while `timeStart < now < timeEnd`.
  - This plays head SFX once (AudioManager de-dupe by `timeStart`) and sets `*(u16*)(hold+0x5C)=0x0101`.
- Skip arc traces/void arcs (`note+0x9C != 0`) so they do not emit particles or get judged.

`TouchLike` fields used in the synthetic stub:

- `touch+0x0C` system id (int)
- `touch+0x1C` ndc_x (float)
- `touch+0x20` ndc_y (float)
- `touch+0x34` touch_uid (int)
- `touch+0x3C` timestamp (int)
